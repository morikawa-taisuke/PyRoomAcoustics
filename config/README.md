# データセット生成用 Config マニュアル

このドキュメントは、音源強調データセット生成時に使用する設定ファイル（`config/dataset_config.yaml`）の各項目の説明と設定方法について解説します。

---

## 全体構造

設定ファイル（YAML形式）は、大きく分けて以下の4つのセクションで構成されています。

1. **`paths`**: 入出力に関するディレクトリ・ファイルパスの設定
2. **`room_params`**: 部屋の寸法や音源（話者、雑音、マイク）の位置に関する空間設定
3. **`acoustic_params`**: 残響時間（Rt60）や SNR などの音響パラメータ設定
4. **`system`**: マルチプロセス数やサンプリング周波数などのシステム設定

---

## 1. `paths` セクション

データセットの入出力パスや、実行対象のデータスプリットを定義します。  
**注意:** ここで指定するディレクトリ名（`speech_dir` 等）の起点は、プロジェクト内の `src/mymodule/const.py` で定義されているベースパス（`SOUND_DATA_DIR` や `SAMPLE_DATA_DIR`）となります。ディレクトリの「絶対パス」ではなく、「ベースパスからの相対パス（またはフォルダ名）」を指定してください。

```yaml
paths:
  # 処理対象のデータスプリットのリスト
  # ここで指定した要素名のサブディレクトリが入力側で検索され、出力側にも作成されます。
  # 例: "train" 実行時、speech_dir/train の中のWAVファイルが処理対象となります。
  splits:
    - train
    - val
    - test
  
  # 音声データ（クリーン）が置かれているディレクトリ
  speech_dir: "speaker/VCTK"
  
  # 雑音データが置かれているディレクトリ
  # ※ここに置かれているWAVファイルからランダムに1波形が選出され、重畳されます。
  noise_dir: "noise/DEMAND_16kHz"
  
  # 生成されたデータセット全体を保存するルートディレクトリ
  # ここに各split名のフォルダと、最終的な dataset.csv が出力されます。
  output_dir: "mix_data"
```

## 2. `room_params` セクション

シミュレーション（Image Source Method）で構築する「部屋のサイズ」と、それに伴う「音源・マイク空間座標」を三次元ベクトル `[x, y, z]`（単位：メートル）で設定します。

```yaml
room_params:
  # 部屋の寸法 (横, 奥行き, 高さ)
  # ※ 大きすぎる値極端な比率を設定すると、計算時（pyroomacoustics内部）でエラーや極端な音割れが発生しやすいため、
  # 当面は [5.0, 5.0, 5.0] のような一般的な固定値を推奨します。
  dimensions: [5.0, 5.0, 5.0]
  
  # マイクロフォンの位置 (x, y, z)
  # ※ 部屋の中心(部屋寸法÷2)からの相対位置ではなく、原点(0,0,0)からの絶対座標で指定します。
  mic_position: [2.5, 2.5, 2.5]
  
  # 話者（クリーン音声）の位置 (x, y, z)
  speech_position: [1.5, 2.5, 1.5]
  
  # 雑音源の位置 (x, y, z)
  noise_position: [3.5, 2.5, 3.5]
```

## 3. `acoustic_params` セクション

音声に付加するシミュレーションパラメータ（SNR および 残響時間 Rt60）を設定します。  
`min` と `max` を設定することで、生成時にその範囲内の「一様乱数（ランダム値）」として適用されます。
**毎回同じ条件（固定値）で生成したい場合は、`min` と `max` に同じ数値を設定してください。**

```yaml
acoustic_params:
  # SNR (Signal-to-Noise Ratio: 信号対雑音比) [dB]
  # 値が小さい（0に近い・マイナス）ほど雑音が大きく、値が大きいほど雑音が小さくなります。
  snr:
    min: 0.0
    max: 10.0
    # 固定値(例: 5dB固定)にしたい場合: min: 5.0, max: 5.0
    
  # 残響時間 (Rt60) [秒]
  # 音のエネルギーが60dB減衰するまでにかかる時間。大きいほどお風呂場のような強いエコーがかかります。
  rt60:
    min: 0.2
    max: 0.5
    # 固定値(例: 0.3秒固定)にしたい場合: min: 0.3, max: 0.3
```

## 4. `system` セクション

スクリプト全体に関わるハードウェア・環境設定を行います。

```yaml
system:
  # 保存される最終的な音声ファイルのサンプリング周波数 [Hz]
  # 入力ファイルもこの周波数であることが前提となります。（異なる場合はスクリプトでエラーが発生します）
  sample_rate: 16000
  
  # マルチプロセスの並列ワーカー数
  # スクリプト実行時の処理速度に直結します。
  # ご使用のPCのCPUコア数を目安に指定してください。（多すぎるとメモリ不足になる可能性があります）
  # 例: 8コアCPUなら 6〜8 程度が推奨です。
  num_workers: 6
```

---

## 💡 YAML編集時の注意点
* インデント（字下げ）は**半角スペース**を使用してください（Tabキーはエラーになります）。
* パスを指定する文字列は、エラー防止のためダブルクォーテーション `"` で囲うことを推奨します。
* リスト（複数指定）は `- `（ハイフンとスペース）を用いて記述するか、配列 `[要素1, 要素2]` の形で記述します。
