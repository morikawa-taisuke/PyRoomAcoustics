# 設定ファイル (`dataset_config.yaml`) の使い方

このドキュメントでは、データセット生成のすべての挙動を制御する `config/dataset_config.yaml` の各パラメータについて説明します。

---

## 1. `paths` (ディレクトリ・データ分割設定)

入出力のフォルダや、処理するデータセットの分割（Train / Val / Test など）を指定します。

*   **`splits`**: 処理対象となるサブフォルダのリストです。例えばここに `[train, val]` と書くと、音声フォルダ内の `train` と `val` ディレクトリにある音声のみが処理されます。
*   **`speech_dir`**: クリーンな音声（話者）が保存されているディレクトリ名です。このパスは `src/core/const.py` にある `SPEECH_DATA_DIR` からの相対パスとして扱われます。
*   **`noise_dir`**: ノイズデータが保存されているディレクトリ名です。同様に `NOISE_DATA_DIR` からの相対パスとして扱われます。
*   **`output_dir`**: 生成された4種類の波形とCSVが保存される最終出力ディレクトリ名です。`MIX_DATA_DIR` からの相対パスとなります。

---

## 2. `room_params` (部屋と配置のシミュレーション設定)

PyRoomAcoustics でシミュレーションされる空間（靴箱型の部屋）の形状や、マイク・音源の配置座標を指定します。
単位はすべて **メートル [m]** であり、部屋の角 `(0, 0, 0)` を原点とする絶対座標で指定します。

*   **`dimensions`**: 部屋の寸法（幅, 奥行き, 高さ）です。例: `[5.0, 5.0, 5.0]`
*   **`mic_position`**: マイクロフォン（録音機材）の配置座標です。
*   **`speech_position`**: 話者（目的の音声）の発生源の座標です。
*   **`noise_position`**: ノイズの発生源の座標です。

> **💡 ヒント**: 座標が部屋の寸法(`dimensions`)の外に出てしまうとシミュレーションエラーになるためご注意ください。

---

## 3. `acoustic_params` (音響特性の設定)

生成データの難易度を決める重要なセクションです。SNR（音圧比）と残響時間（RT60）を指定します。
それぞれ `min` (最小値) と `max` (最大値) を指定することで、音声ファイルごとに **ランダムな値** が選ばれ、データセットに多様性を持たせることができます。

*   **`snr` (Signal-to-Noise Ratio)**: 信号と雑音の比率です（単位: dB）。
    *   値が **大きい** ほどノイズが小さく聞き取りやすい音声になります。
    *   値が **小さい**（あるいはマイナス）ほどノイズが大きく過酷な環境になります。
*   **`rt60`**: 残響時間（Reverberation Time）です（単位: 秒）。
    *   60dB減衰するまでにかかる時間のことです。値が大きいほど「お風呂場のような」強いエコーがかかります。

> **💡 固定値で生成したい場合**
> ランダムではなく常に同じ条件（例: 常に SNR 5.0dB、RT60 0.3秒）で生成したい場合は、以下のように `min` と `max` に全く同じ値を設定してください。
> ```yaml
> acoustic_params:
>   snr:
>     min: 5.0
>     max: 5.0
>   rt60:
>     min: 0.3
>     max: 0.3
> ```

---

## 4. `system` (システム動作設定)

スクリプトの動作環境に関する設定です。

*   **`sample_rate`**: 入出力のサンプリング周波数（Hz）です。基本は `16000` (16kHz) 固定での運用をおすすめします。扱う音声データも全てこの周波数に揃っている必要があります。
*   **`num_workers`**: マルチプロセス処理の並列実行数です。お使いのPCのCPUコア数に合わせて大きくすると、生成速度が劇的に向上します。（例: 6コアCPUなら `6` を指定すると快適です）
